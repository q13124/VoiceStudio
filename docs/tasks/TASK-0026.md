# TASK-0026: Introduce Engine Interface Layer

## Objective

Introduce a formal engine interface layer (IEnginePort) to establish clean architecture boundaries between backend services and engine implementations, following Ports & Adapters pattern.

## Affected Modules

- [x] `backend/interfaces/__init__.py` — Interface layer initialization
- [x] `backend/interfaces/engine_port.py` — IEnginePort and engine-specific protocols
- [x] `backend/adapters/__init__.py` — Adapter layer initialization
- [x] `backend/adapters/engine_adapter.py` — Engine adapter implementations
- [x] `backend/services/engine_service.py` — get_engine_port integration

## Constraints

- Must follow Ports & Adapters (Hexagonal) architecture
- No breaking changes to existing engine consumers
- Protocol-based typing for static analysis support

## Required Proofs

- [x] Interface layer created with proper abstractions
- [x] Adapter implementations compile
- [x] Existing engine consumers continue to function

## Acceptance Criteria

- [x] IEnginePort protocol defined
- [x] Engine-specific protocols (ISynthesisEngine, ITranscriptionEngine, etc.)
- [x] EngineAdapter class implementing IEnginePort
- [x] Integration with engine_service.py
- [x] DTOs for engine operations
