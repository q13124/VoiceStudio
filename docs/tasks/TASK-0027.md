# TASK-0027: Accessibility Enhancements

**Created**: 2026-02-02  
**Owner**: Role 3 (UI Engineer)  
**Priority**: Medium  
**Phase**: 6+ (Optional Enhancements)

---

## Objective

Add accessibility support (screen reader, ARIA, focus indicators) to all interactive controls across VoiceStudio views. Currently only 7/95 views have `AutomationProperties.Name` set.

---

## Problem Statement

| Metric | Current | Target |
|--------|---------|--------|
| Views with AutomationProperties | 7/95 (7%) | 95/95 (100%) |
| AutomationProperties.Name usage | 77 instances | ~500+ instances |

Enterprise adoption and accessibility compliance require screen reader support for all interactive controls.

---

## Affected Modules

- [ ] `src/VoiceStudio.App/Views/Panels/*.xaml` - 88 views need accessibility properties
- [ ] `src/VoiceStudio.App/Views/Shell/*.xaml` - Navigation and shell elements
- [ ] `src/VoiceStudio.App/Views/Dialogs/*.xaml` - Dialog accessibility

---

## Acceptance Criteria

- [ ] All Button, TextBox, ComboBox, Slider controls have `AutomationProperties.Name`
- [ ] All interactive controls have `AutomationProperties.HelpText` where appropriate
- [ ] Tab navigation order is logical (verified with keyboard-only testing)
- [ ] Focus indicators are visible for all focusable elements
- [ ] Screen reader testing passes (Narrator or NVDA)

---

## Implementation Plan

### Phase 1: Core Panels (4h)
Add AutomationProperties to the 6 core panels:
- Profiles (Left)
- Timeline (Bottom)
- EffectsMixer (Center)
- Analyzer (Right)
- Macro (Center)
- Diagnostics (Bottom)

### Phase 2: High-Use Panels (4h)
- VoiceCloningWizard
- Settings
- QualityControl
- RealTimeVoiceConverter

### Phase 3: Remaining Panels (4-8h)
- All other panels in Views/Panels/

### Phase 4: Shell & Dialogs (2h)
- NavigationView
- All dialogs
