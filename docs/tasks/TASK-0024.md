# TASK-0024: Fix VS-0035 XAML Compiler Silent Failure

## Objective

Resolve the XAML compiler (XamlCompiler.exe) silent failure that causes C# builds to fail with exit code 1 and no error output. This is a regression blocker that prevents all C# development.

## Affected Modules

- [ ] `tools/xaml-compiler-wrapper.cmd` - Path resolution and logging fixes applied
- [ ] `Directory.Build.props` - SDK version configuration
- [ ] `src/VoiceStudio.App/` - All WinUI/XAML compilation affected

## Root Cause Analysis

### Investigation Summary (2026-01-30)

1. **Wrapper Script Issues (FIXED)**:
   - Delayed expansion variables (`%RAW_LOG%` → `!RAW_LOG!`) fixed
   - Double backslash normalization moved before path existence checks
   - Raw logging now captures compiler invocation

2. **SDK Version Mismatch**:
   - `Directory.Build.props` specifies `1.8.251106002`
   - Locally installed WinUI SDK is `1.8.251105000`
   - These are different packages (`microsoft.windowsappsdk` vs `microsoft.windowsappsdk.winui`)
   - NuGet cache has locked DLLs preventing update

3. **XamlCompiler.exe Silent Failure**:
   - Compiler runs for ~5 seconds then exits with code 1
   - No stdout/stderr output captured
   - No output.json generated
   - Same behavior with or without wrapper

4. **40,567 Errors on XamlPreCompile Target**:
   - Errors indicate basic .NET types (`System.String`, `System.Object`) not found
   - Suggests reference assembly resolution failure in XAML compiler
