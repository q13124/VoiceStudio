# TASK-0029: Advanced Panel Backend Integration

**Created**: 2026-02-02  
**Owner**: Role 3 (UI Engineer)  
**Priority**: Medium  
**Phase**: 6+ (Optional Enhancements)

---

## Objective

Wire backend APIs for 9 innovative panels that currently have UI scaffolding but lack backend integration:

1. Text Speech Editor
2. Prosody Control
3. Spatial Audio
4. AI Mixing & Mastering
5. Ensemble Synthesis
6. Voice Morph
7. Quality Dashboard
8. Profile Health Dashboard
9. Automation

---

## Problem Statement

These panels have XAML views and ViewModels but:
- Backend API calls are stubbed or missing
- Data is mocked or static
- User actions don't persist or trigger synthesis

## Acceptance Criteria

Per panel:
- [ ] ViewModel calls real backend APIs (not mocked)
- [ ] Loading states shown during API calls
- [ ] Error states handled gracefully
- [ ] Data persists across panel switches
- [ ] No runtime binding failures

---

## Implementation Plan

### Phase 1: API Audit (4h)
For each panel:
1. List required backend endpoints
2. Verify endpoints exist in OpenAPI spec (508 paths)
3. Identify missing endpoints

### Phase 2: BackendClient Extensions (8h)
Add missing methods to `BackendClient.cs`:
- Prosody control endpoints
- Spatial audio endpoints
- Quality metrics endpoints
- etc.

### Phase 3: ViewModel Integration (24h)
Per panel (~2-3h each):
1. Replace mock data with API calls
2. Add loading/error states
3. Test functionality

### Phase 4: Testing (8h)
- Verify each panel works with backend running
- Test error handling (backend down)
- Test data persistence
