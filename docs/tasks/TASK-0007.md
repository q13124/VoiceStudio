# TASK-0007: SLO Dashboard Panel

## Objective

Create a WinUI 3 panel for monitoring Service Level Objectives (SLOs), displaying current status, violation history, and trend charts using the existing `/api/telemetry/slo/*` endpoints.

## Affected Modules

- [ ] `src/VoiceStudio.App/Views/SLODashboardPanel.xaml` - New panel XAML
- [ ] `src/VoiceStudio.App/Views/SLODashboardPanel.xaml.cs` - Code-behind
- [ ] `src/VoiceStudio.App/ViewModels/SLODashboardViewModel.cs` - ViewModel for panel
- [ ] `src/VoiceStudio.App/Services/BackendClient.cs` - Add SLO API methods
- [ ] `src/VoiceStudio.Core/Panels/PanelDescriptor.cs` - Register new panel

## Constraints

- Must use VSQ.* design tokens only (no hardcoded colors)
- Must integrate with existing PanelHost system
- Must use MVVM pattern (no code-behind logic)
- Charts must render efficiently (no UI blocking)
- Auto-refresh must be cancellable and configurable

## Required Proofs

- [ ] Build log (success)
- [ ] UI smoke test (panel loads without binding errors)
- [ ] Screenshot showing 6 SLO cards
- [ ] Violation history displaying correctly

## Acceptance Criteria

- [ ] Panel appears in panel navigation/selection
- [ ] All 6 SLOs displayed as status cards
- [ ] Each card shows: name, target, current value, status (green/yellow/red)
- [ ] Violation history timeline visible
- [ ] Trend charts show historical data (24h, 7d, 30d selectable)
- [ ] Auto-refresh with configurable interval (default 30s)
- [ ] Zero binding errors at runtime

## Features

### 6 SLO Cards

| SLO | Target | Measurement |
|-----|--------|-------------|
| SLO-1 | Voice Synthesis Latency | < 3s P50 |
| SLO-2 | Transcription Throughput | > 10x realtime |
| SLO-3 | API Response Time | < 100ms health, < 200ms CRUD |
| SLO-4 | UI Responsiveness | < 5s startup |
| SLO-5 | Engine Availability | > 99% uptime |
| SLO-6 | Voice Cloning Quality | MOS > 3.5, Similarity > 0.7 |

### Card Design

```
┌─────────────────────────────┐

│ ────────────────────────────│

│ Current: 2.1s               │
│ Status: ● MEETING           │
│ [Trend Chart]               │
└─────────────────────────────┘
```

### API Integration

Existing endpoints:
- `GET /api/telemetry/slo` - All SLO statuses
- `GET /api/telemetry/slo/{slo_id}` - Specific SLO with history
- `GET /api/telemetry/metrics` - Prometheus-format metrics
