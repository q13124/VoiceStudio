# ADR-013: OpenTelemetry Distributed Tracing

## Status

Proposed

## Context

VoiceStudio has multiple components (UI, backend, engines). Distributed tracing is needed for debugging and performance monitoring.

## Options Considered

1. **No tracing** - Rely on logs
2. **OpenTelemetry** - Standard observability framework
3. **Vendor-specific** - Datadog, Sentry, etc.

## Decision

**To be decided.** This ADR is a placeholder.

Implementation evidence:
- Prometheus metrics instrumentation
- SLO tracking in synthesis/transcription
