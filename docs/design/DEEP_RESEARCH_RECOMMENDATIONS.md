# Deep Research Recommendations

## When to Use Deep Research

**Use Deep Research when:**
- You need current best practices for a technology
- You need example code for complex integrations
- You need to understand performance implications
- You need to compare multiple approaches
- You need to understand framework-specific patterns

**Don't use Deep Research for:**
- Simple implementations you can do directly
- Well-documented standard patterns
- Things already covered in your specs

---

## Recommended Deep Research Topics

### 1. **Drag-and-Drop in WinUI 3** ⭐⭐⭐
**When:** Before implementing drag-and-drop features

**Prompt:**
```
Research best practices and example code for implementing drag-and-drop functionality in WinUI 3 (.NET 8) desktop applications. Focus on:
- Drag-and-drop between panels
- Drag audio files into timeline
- Visual drag feedback
- Performance considerations
- Accessibility support
Provide minimal working code examples for common drag-drop scenarios.
```

**Why:** Drag-and-drop is complex and WinUI 3 has specific patterns

### 2. **Accessibility in WinUI 3** ⭐⭐⭐
**When:** Before implementing accessibility features

**Prompt:**
```
Research comprehensive accessibility implementation for WinUI 3 (.NET 8) desktop applications. Cover:
- Screen reader integration (Narrator)
- Keyboard-only navigation patterns
- High contrast mode support
- Automation properties best practices
- ARIA-like patterns for WinUI 3
- Testing accessibility features
Provide code examples for common accessibility scenarios.
```

**Why:** Accessibility is critical and has specific WinUI 3 requirements

## How to Use Deep Research

1. **Identify the specific problem** - What exactly do you need to know?
2. **Formulate a clear prompt** - Include context, requirements, and desired output
3. **Review the research** - Extract relevant patterns and code
4. **Adapt to your architecture** - Integrate findings into your existing structure
5. **Test and iterate** - Verify the approach works in your context
