## 📋 Tracking Systems You MUST Use

### 1. **TASK_LOG.md** - Central Task Log
- **Location:** `docs/governance/TASK_LOG.md`
- **When to Update:** 
  - ✅ BEFORE starting work (check assigned tasks)
  - ✅ AFTER completing work (update status to ✅ Complete)
- **How to Update:** Change status column in the task table

### 2. **TASK_TRACKER_3_WORKERS.md** - Detailed Progress Tracker
- **Location:** `docs/governance/TASK_TRACKER_3_WORKERS.md`
- **When to Update:** 
  - ✅ DAILY (check off completed tasks with `[x]`)
  - ✅ Update progress percentages
- **How to Update:** Change `[ ]` to `[x]` for completed tasks

### 3. **MASTER_TASK_CHECKLIST.md** - Master Checklist
- **Location:** `docs/governance/MASTER_TASK_CHECKLIST.md`
- **When to Update:** 
  - ✅ AFTER completing tasks (mark as ✅ COMPLETE)
  - ✅ BEFORE starting (check if already complete)
- **How to Update:** Change status to ✅ COMPLETE

### 4. **Your Worker Status File** - Individual Status Report
- **Location:** `docs/governance/worker[N]/WORKER_[N]_STATUS_2025-01-28.md`
- **When to Update:** 
  - ✅ DAILY (update progress status)
- **How to Update:** Update your status file with current progress

## ✅ Required Daily Workflow

### Every Day Before Starting Work:
1. ✅ Check `TASK_LOG.md` - See what tasks are assigned to you
2. ✅ Check `TASK_TRACKER_3_WORKERS.md` - See your current progress
3. ✅ Check `MASTER_TASK_CHECKLIST.md` - Verify tasks aren't already complete

### Every Day After Completing Work:
1. ✅ Update `TASK_LOG.md` - Change status to ✅ Complete
2. ✅ Update `TASK_TRACKER_3_WORKERS.md` - Mark tasks complete `[x]`
3. ✅ Update `MASTER_TASK_CHECKLIST.md` - Mark as ✅ COMPLETE
4. ✅ Update your Worker Status file - Document completion
