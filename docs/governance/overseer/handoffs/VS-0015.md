## Summary

Verified and hardened the `ProjectStoreService` storage migration logic. Gate D requires "Storage migrations idempotent, versioned, and tested".

---

## Changes

### `backend/services/ProjectStoreService.py`

- Updated `ProjectRecord` Pydantic model to allow `schema_version >= 0` (was >= 1). This permits loading legacy files that default to 0.
- Updated `_load_record_from_dir` to explicitly detect missing `schema_version` in raw JSON and force it to `0` before validation.
- Updated `_migrate_record` to handle `v0 -> v1` upgrade.

### `tests/unit/backend/services/test_project_store_migration.py` (New)

- Added tests for:
  - Loading a legacy v0 project (missing version field).
  - Verifying migration updates the file on disk.
  - Verifying idempotency (loading twice doesn't change file or data).
  - Verifying invalid versions are rejected.
