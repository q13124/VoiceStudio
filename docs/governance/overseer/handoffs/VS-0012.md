## Key fix

- **Windows App Runtime native DLLs removed from publish output**: `src/VoiceStudio.App/VoiceStudio.App.csproj` includes an `AfterTargets="Publish"` target (`ExcludeSystemDllsFromPublish`) that deletes:
  - `CoreMessagingXP.dll`
  - `dcompi.dll`
  - `dwmcorei.dll`
  - `Microsoft.ui.xaml.dll`
  - `Microsoft.ui.xaml.resources.common.dll`
  - `Microsoft.ui.xaml.resources.19h1.dll`
  - `Microsoft.UI.Xaml.Controls.dll`
  - `Microsoft.UI.Xaml.Controls.pri`
  - `Microsoft.UI.Xaml.Internal.dll`
  - `Microsoft.UI.Xaml.Phone.dll`
  - `Microsoft.UI.Xaml.winmd`
  - `Microsoft.UI.pri`

This prevents the published unpackaged EXE from crash/fail-fast modes observed when those DLLs were bundled as loose files in the publish directory.

### XAML payload in publish output (XBF)

Published unpackaged artifacts must include compiled XAML payload (`*.xbf`). If missing, WinUI will fail early with errors like:

- `XamlParseException: Cannot locate resource from 'ms-appx:///MainWindow.xaml'`

Fix: `src/VoiceStudio.App/VoiceStudio.App.csproj` now includes an `AfterTargets="Publish"` target `IncludeXbfInPublishOutput` that copies XBF from `$(TargetDir)` into `$(PublishDir)` so `ms-appx:///...` resource resolution works from the published folder.
