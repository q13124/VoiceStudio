## Summary

Implemented the `EngineManager` service and `BackendEngineAdapter` to support the Gate E requirement for pluggable engine integration.
This service discovers engines from the backend and exposes them via the standardized `IEngine` interface to the UI.

---

## Changes

### `src/VoiceStudio.App/Services/EngineManager.cs` (New)

- Central service to manage engine lifecycle and discovery.
- Initializes by querying `GET /api/engines` (via `IBackendClient`).
- Creates `BackendEngineAdapter` instances for discovered engines.
- Heuristic capability detection based on engine ID (temporary until backend metadata endpoint is fully standardized).

### `src/VoiceStudio.App/Core/Engines/BackendEngineAdapter.cs` (New)

- Implements `IEngine`, `ITextToSpeechEngine`, `ITranscriptionEngine`.
- Proxies calls to the Python backend via `IBackendClient`:
  - `InitializeAsync` -> `POST /api/engines/{id}/start`
  - `ShutdownAsync` -> `POST /api/engines/{id}/stop`
  - `IsAvailableAsync` -> `GET /api/engines/{id}/status`
  - `GetVoicesAsync` -> `GET /api/engines/{id}/voices`
  - `SynthesizeAsync` -> `POST /api/voice/synthesize`
  - `TranscribeAsync` -> `POST /api/transcribe`

### `backend/api/routes/engines.py`

- Added lifecycle endpoints:
  - `POST /start`
  - `POST /stop`
  - `GET /status`
  - `GET /voices`
- These endpoints integrate with `app.core.runtime.engine_lifecycle`.

### `src/VoiceStudio.App/Services/Stores/EngineStore.cs`

- Updated to use `EngineManager` for discovery if available.

### `src/VoiceStudio.App/Services/ServiceProvider.cs`

- Registered `EngineManager` in the DI container.
