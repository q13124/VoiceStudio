## Summary

Implemented content-addressed audio cache to deduplicate waveforms and model artifacts, fulfilling Gate D requirement. Audio files are now stored by SHA-256 hash in a cache directory, and project audio saves reference the cached copy instead of creating duplicate files.

---

## Changes

### New Service: `ContentAddressedAudioCache`

**File:** `backend/services/ContentAddressedAudioCache.py`

- SHA-256 hash-based storage for audio files
- Directory sharding by hash prefix (first 2 hex characters)
- Cache structure: `<cache_root>/audio/<prefix>/<hash>.wav`
- Thread-safe operations with `threading.RLock`
- Singleton accessor pattern: `get_audio_cache()`
- Cache root defaults to `~/.voicestudio/cache` (overridable via `VOICESTUDIO_CACHE_DIR`)

**Key methods:**
- `get_or_store(source_path)` — computes hash, returns cached path (stores if missing)
- `get_by_hash(hash_value)` — retrieves cached file by hash
- `ensure_cached(source_path)` — convenience wrapper

### Integration: Project Audio Save

**File:** `backend/api/routes/projects.py`

- Modified `save_audio_to_project` to use content-addressed cache
- Flow: source → compute hash → get/store in cache → copy from cache to project
- Graceful fallback to direct copy if cache fails

### Test Coverage

**File:** `tests/unit/backend/api/routes/test_projects.py`

- Added `test_content_addressed_cache_deduplication`
- Verifies identical audio files saved to multiple projects reference the same cached copy
