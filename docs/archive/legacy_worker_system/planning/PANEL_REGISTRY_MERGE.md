## 🔗 Manual Registry Merge

The manual `PanelRegistry.cs` must merge auto-discovered panels:

```csharp
using System.Collections.Generic;
using System.Linq;

namespace VoiceStudio.Core.Panels {

  public class PanelRegistry : IPanelRegistry {
    
    private readonly List<PanelDescriptor> _panels = new();
    
    public static void RegisterAll() {
      var registry = new PanelRegistry();
      
      // 1. Register auto-discovered panels
      foreach (var xamlPath in PanelRegistryAuto.AllXaml()) {
        registry.RegisterPanelFromXaml(xamlPath);
      }
      
      // 2. Register hand-picked defaults
      registry.RegisterDefaultPanel("Profiles", PanelRegion.Left);
      registry.RegisterDefaultPanel("Timeline", PanelRegion.Center);
      registry.RegisterDefaultPanel("EffectsMixer", PanelRegion.Right);
      registry.RegisterDefaultPanel("Macro", PanelRegion.Bottom);
      
      // 3. Set default layout
      registry.SetDefaultLayout();
      
      return registry;
    }
    
    private void RegisterPanelFromXaml(string xamlPath) {
      // Extract panel name from path
      var panelName = ExtractPanelName(xamlPath);
      
      // Check if ViewModel exists
      var viewModelPath = GetViewModelPath(panelName);
      if (!File.Exists(viewModelPath)) {
        // Scaffold stub ViewModel
        ScaffoldViewModel(panelName);
        Logger.LogWarning($"TODO: Implement {panelName}ViewModel");
      }
      
      // Register panel
      var descriptor = new PanelDescriptor {
        PanelId = panelName,
        DisplayName = FormatDisplayName(panelName),
        Region = DetermineRegion(panelName),
        ViewType = typeof(UserControl), // Will be resolved at runtime
        ViewModelType = GetViewModelType(panelName),
        Tier = DetermineTier(panelName),
        Category = DetermineCategory(panelName)
      };
      
      _panels.Add(descriptor);
    }
    
    private void ScaffoldViewModel(string panelName) {
      var viewModelCode = $@"
using CommunityToolkit.Mvvm.ComponentModel;
using CommunityToolkit.Mvvm.Input;

namespace VoiceStudio.App.ViewModels.Panels {{
  public sealed partial class {panelName}ViewModel : ObservableObject {{
    // TODO: Implement {panelName}ViewModel
    public {panelName}ViewModel() {{
      // No-throw stub
    }}
  }}
}}
";
      // Write ViewModel file
      var path = GetViewModelPath(panelName);
      Directory.CreateDirectory(Path.GetDirectoryName(path));
      File.WriteAllText(path, viewModelCode);
    }
  }

}
```

## ✅ Strict Rules

### 1. Load ALL Panels

- ✅ **MUST** include all panels from `PanelRegistryAuto.AllXaml()`
- ✅ **MUST** not skip any panels
- ✅ **MUST** not hide "unused" panels
- ✅ **MUST** register every discovered panel

### 2. Missing ViewModels

- ✅ **MUST** scaffold stub ViewModel if missing
- ✅ **MUST** log TODO warning
- ✅ **MUST** use no-throw stub (no exceptions)
- ✅ **MUST** allow panel to load even without full ViewModel

### 3. Panel Count

- ✅ **MUST** reflect ~90+ panels after merge
- ✅ **MUST** verify count matches discovery
- ✅ **MUST** log panel count for verification

## 🔧 Implementation Pattern

### Step 1: Auto-Discovery

```csharp
// Generated by migration script
var autoPanels = PanelRegistryAuto.AllXaml();
// Returns: ["ui/Views/Panels/Panel1View.xaml", ...]
```

### Step 2: Registration

```csharp
foreach (var xamlPath in autoPanels) {
  // Extract panel info
  var panelId = ExtractPanelId(xamlPath);
  var viewModelExists = CheckViewModelExists(panelId);
  
  // Scaffold if missing
  if (!viewModelExists) {
    ScaffoldViewModel(panelId);
  }
  
  // Register panel
  RegisterPanel(panelId, xamlPath);
}
```

### Step 3: Defaults

```csharp
// Set hand-picked defaults
SetDefaultPanel("Profiles", PanelRegion.Left);
SetDefaultPanel("Timeline", PanelRegion.Center);
// ...
```

### Step 4: Verification

```csharp
var totalPanels = GetPanelCount();
if (totalPanels < 90) {
  Logger.LogWarning($"Panel count ({totalPanels}) lower than expected (~90+)");
}
```

## 🚨 Common Issues

### Issue: Panel Not Found

**Solution:**
- Check `PanelRegistryAuto.AllXaml()` includes the panel
- Verify panel is in one of the searched directories
- Re-run migration script to regenerate auto-list

### Issue: ViewModel Missing

**Solution:**
- Stub ViewModel should be auto-scaffolded
- Check scaffolded file exists
- Verify no-throw stub allows panel to load

### Issue: Panel Count Mismatch

**Solution:**
- Compare `PanelRegistryAuto.AllXaml().Count()` with registered count
- Check for deduplication issues
- Verify all panels are being registered
