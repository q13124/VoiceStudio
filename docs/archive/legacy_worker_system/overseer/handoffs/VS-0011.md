# VS-0011 Handoff — ServiceProvider recursion fix

**Date:** 2025-01-28  
**Role:** Core Platform Engineer  
**Gate:** C  
**State:** DONE

---

## Summary

Fixed a `StackOverflowException` during application startup caused by recursive calls in `ServiceProvider.Initialize()`.

---

## Changes

### File: `src/VoiceStudio.App/Services/ServiceProvider.cs`

- Introduced `private static bool _isInitializing` flag.
- Added guard clause in `Initialize()`: `if (_initialized || _isInitializing) return;`.
- Wrapped initialization logic in `try/finally` to ensure `_isInitializing` is reset.

### Logic

- Previous behavior: `Initialize()` -> Service Ctor -> `GetErrorLoggingService()` -> `if(!_initialized) Initialize()` -> Loop.
- New behavior: `Initialize()` -> Service Ctor -> `GetErrorLoggingService()` -> `if(_isInitializing) return` -> Returns existing (or null) instance without recursion.
