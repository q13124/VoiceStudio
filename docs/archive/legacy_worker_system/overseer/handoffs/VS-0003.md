## Summary

- **Goal**: Prove installer lifecycle end-to-end: install → launch → upgrade → rollback → uninstall.
- **Result**: PASS for v1.0.0 ↔ v1.0.1 with deterministic logs under `C:\logs\`.
- **Key fix (blocking root cause)**: Unpackaged WinUI 3 requires `.pri` resource indexes alongside the app.
  - Missing PRI files caused: `XamlParseException: Cannot locate resource from 'ms-appx:///Microsoft.UI.Xaml/Themes/themeresources.xaml' (0x802B000A)` during installed launch.
  - Fix implemented in `scripts/gatec-publish-launch.ps1`: publish now copies required PRI files into the Gate C publish folder:
    - `Microsoft.UI.pri`
    - `Microsoft.UI.Xaml.Controls.pri`
    - `Microsoft.WindowsAppRuntime.pri`
    - `VoiceStudio.App.pri`
  - `installer/build-installer.ps1` now treats these PRI files as required publish inputs (won’t reuse an incomplete publish folder).
